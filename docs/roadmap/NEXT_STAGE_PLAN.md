# 下一阶段工作计划

## 目标

把本地知识库从“资料已处理”推进到“可稳定用于日常工作的参谋系统”。

## 第一阶段：GitHub 工程控制面稳定

- Codex：重写 main、清理公开边界、决定旧内容归档或删除。
- MiniMax：审读 README、AGENTS、公开边界和工单模板。
- 本地模型：不参与。
- cminimax：仅在 Codex 额度不足时，按工单生成候选文档。

验收：main 中不含原始资料、OCR 全文、向量、索引、密钥。

## 第二阶段：本地参谋检索入口

- Codex：设计统一查询入口，决定关键词、向量、OCR 临时层的路由规则。
- MiniMax：整理 20-30 个真实工作问题和预期命中材料。
- 本地模型：不做判断，只提供已完成向量和 OCR 层。
- cminimax：额度不足时，生成评测题候选和失败样例摘要。

验收：常见工作问题能返回材料、出处、可信度标签。

## 第三阶段：上海盟讯引用级 OCR 审校

- Codex：制定引用级 OCR 标准、元数据修正规则和人工审校流程。
- MiniMax：按页审读候选 OCR，输出疑似错字、版面风险、元数据异常。
- 本地模型：继续 OCR、版面抽取、文本分块。
- cminimax：额度不足时，只做批量审校候选，不做定稿。

验收：至少一批样本页达到可引用标准，且有原图定位。

## 第四阶段：2021 档案人物字段核验

- Codex：定义人物字段 schema 和入库门槛。
- MiniMax：逐个 PDF/OCR 对照抽字段候选。
- 本地模型：负责 OCR 和图片/PDF 预处理。
- cminimax：额度不足时，按人物生成字段候选表。

验收：字段分为 verified / candidate / missing，不把 OCR 猜测写成定稿。

## 第五阶段：外部研究网站接入

- Codex：定义来源分级、版权边界和引用策略。
- MiniMax：整理海外一手研究平台清单和入口摘要。
- 本地模型：后续可向量化公开摘要或自建摘录。
- cminimax：额度不足时，生成来源候选表。

验收：外部资料只作为来源入口和公开摘要，不混入内部定稿层。
