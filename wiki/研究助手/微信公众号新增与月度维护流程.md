# 微信公众号新增与月度维护流程

生成时间：2026-06-17

## 目标

把新增公众号、现有公众号新增文章、分类校订、样本更新、Obsidian 同步和 GitHub 提交固定成月度流程，保证语料库长期可用。

## 新增公众号流程

1. 把新公众号导出的文章放入 `~/Downloads/微信公众号/公众号名称/`。
2. 先扫描：

```bash
kb scan --input ~/Downloads/微信公众号
```

3. 再导入或刷新：

```bash
kb refresh
```

4. 检查新增数量、失败文件和重复文章。
5. 运行语料库体检：

```bash
kb corpus
kb corpus-audit
kb corpus-style
```

6. 对新公众号先不直接纳入写作模仿对象，先判断它属于主语料、参考语料还是外部材料。

## 现有公众号新增文章流程

1. 把新增文章继续放入原公众号文件夹。
2. 执行 `kb refresh`。
3. 执行 `kb corpus` 和 `kb corpus-style`。
4. 查看 [[微信公众号语料库体检报告]]、[[上海民盟2023年以来写作样本库]] 是否更新。
5. 需要时运行 `/稿` 或 `/数` 验证新文章能被检索。

## 月度维护流程

每月固定执行：

```bash
kb refresh
kb corpus
kb corpus-audit
kb corpus-style
kb external-sources --save
kb guardrails --save
kb verify --save
kb obsidian-sync --vault "/Users/cheer/Library/Mobile Documents/iCloud~md~obsidian/Documents/MllmWiki"
kb obsidian-status --vault "/Users/cheer/Library/Mobile Documents/iCloud~md~obsidian/Documents/MllmWiki" --save
KB_PUBLIC=1 python3 webapp/build_static.py
```

最后执行 Git 提交和推送。

## 人工校订流程

1. 打开 `index/corpus/classification_priority_review.csv`。
2. 只填写人工真正看过的行。
3. 若分类错误，填写 `review_result=错误` 和合法 `suggested_type`。
4. 应用校订：

```bash
kb corpus-apply-reviews --save
```

5. 重新运行 `kb corpus-audit`，确认人工列被保留。

## 收尾检查

- `python3 -B -m unittest discover -s tests`
- `python3 -m compileall -q src tests webapp`
- `kb obsidian-status --vault ".../MllmWiki" --save`
- 检查 `docs/content.json` 不含本机绝对路径。
- `git status --short --branch` 应在提交后干净。

## 风险

- 机器分类不等于人工校订。
- Google Drive 工作文件不自动进入公众号主语料。
- 明文静态站只适合私有仓库或内部预览。
