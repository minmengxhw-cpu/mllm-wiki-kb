# 微信公众号语料库人工校订说明

生成时间：2026-06-11T11:22:39

## 当前校订目标

- 先提高文章主类型准确率，不急于做细颗粒实体抽取。
- 优先校订上海民盟 2023 年以后写作样本。
- 文史/盟史文章只做候选库，不把机器分类当作史实结论。

## 字段说明

- `suggested_type`：人工建议类型代码，如 `activity_report`、`meeting_report`。
- `review_result`：填写 `正确`、`错误`、`不确定`。
- `review_note`：说明误判原因或建议增加/删除的关键词。

## 类型代码

| 类型代码 | 类型名称 |
| --- | --- |
| meeting_report | 会议报道 |
| activity_report | 活动报道 |
| leadership_speech | 领导讲话/工作部署 |
| member_achievement | 盟员履职/成果荣誉 |
| person_profile | 人物采访/人物风采 |
| history_commemoration | 文史纪念 |
| history_research | 盟史研究 |
| policy_advice | 参政议政 |
| theme_education | 主题教育 |
| organization_building | 组织建设 |
| social_service | 社会服务 |
| notice_info | 通知公告/信息发布 |
| commentary_theory | 评论综述/理论文章 |
| other | 其他/待判 |

## 推荐流程

1. 先看 `wiki/研究助手/微信公众号文章分类抽检表.md`，挑误判明显的类型。
2. 再编辑 `classification_review.csv`，保留人工判断。
3. 汇总高频误判词，修改分类规则。
4. 重新运行 `.venv/bin/kb corpus && .venv/bin/kb corpus-audit`。
